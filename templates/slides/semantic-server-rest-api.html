Semantic Server REST API

The semantic server's primary function is providing a REST API that hides the specific implementation details from other applications that may need to interact with BIBFRAME Datastore or other bibliographic and technological systems that extend the Semantic Server REST API. This open-source project is available on Github at https://github.com/jermnelson/semantic-server.

The semantic server's REST API uses an open-source project called Falcon, a Python-based framework for creating cloud-based REST APIs, initially developed and continued sponsorship by Rackspace.

For the Fedora Four repository, the semantic server implements a Resource class with methods that map to the standard GET, POST, PATCH, DELETE HTTP methods. The calling client to retrieve the RDF graph of the resource with a GET call, create a new Fedora container, ingests into Fuseki, and indexed into Elasticsearch with a single POST call, update the RDF graph with a PATCH, and remove the resource with a DELETE http method.
