Participating

The two primary sources from which the Catalog Pull Platform "pulls" requirements are Colorado College and the Library of Congress - two very different organizations. But we want your use cases!

Of course, if you would like to try a more Python-based approach to using Fedora 4, I would love to hear from you! Even better, pull requests are most welcome to these code repositories:

  • BIBFRAME Datastore - https://github.com/jermnelson/BIBFRAME-Datastore
  • bibcat.org - https://github.com/jermnelson/bibframe-catalog
  • Semantic Server https://github.com/jermnelson/semantic-server
  • Islandora OpenBadge - https://github.com/Islandora/Islandora-eBadges-Repository.gites
  • TIGER Cataloger - https://github.com/Tutt-Library/tiger-catalog

One of the biggest pain points at this stage in the development of BIBFRAME Datastore is in installing and getting the various backend components running. (a problem shared with other Fedora-based frontends like Islandora and Hydra) Our goal by the end of the summer to have a Docker-based container to easy some of this issues as well as a standard PIP based installation for both.
