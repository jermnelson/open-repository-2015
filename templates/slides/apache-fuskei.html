Apache Fuskei

A proper SPARQL endpoint is essential for working with Fedora as a Linked Data Platform. The current version of the catalog pull platform uses Apache Foundation Fuskei project for SPARQL queries. The primary use of SPARQL is for ingestion and reporting needs for the project.

For example, when ingesting BIBFRAME Works, Titles, Person, and Organization, we are doing a rough deduplication based on the following entity properties: bf:authorizedAccessPoint, bf:identifierValue, bf:classificationNumber, bf:label, bf:titleValue. We use a SPARQL query similar to this:

PREFIX bf: 
PREFIX rdf: 
PREFIX xsd: 
SELECT ?subject 
WHERE {
   ?subject bf:authorizedAccessPoint "Mark Twain journal."^^xsd:string .
   ?subject rdf:type bf:Work .
}
